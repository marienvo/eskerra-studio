# Security Policy

## Reporting a vulnerability

Please do not report security vulnerabilities through public issues or pull requests.

Instead, send a private report to:

**eskerrasecurity.17vew@passmail.net**

Please include as much relevant information as possible:

- A clear description of the issue
- Steps to reproduce it
- The affected area or feature
- Potential impact
- Any suggested fix, if known

Please avoid publicly disclosing the vulnerability until it has been reviewed and addressed.

## Pull requests for security fixes

Please do not open a pull request for a security fix before reporting the issue privately.

Security-related changes should be discussed privately first, so the fix can be handled carefully.
